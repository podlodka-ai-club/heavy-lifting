# Heavy Lifting

`Heavy Lifting` — это MVP backend-оркестратора для автоматизации работы с задачами, кодом и pull request.

## Что делает проект

Сервис должен:

- забирать задачи из трекера;
- подготавливать контекст для агента;
- запускать выполнение задач, связанных с кодом;
- синхронизировать репозиторий из git;
- создавать ветки и PR через SCM-адаптер;
- забирать комментарии из PR и превращать их в дочерние задачи;
- отправлять результат обратно в трекер;
- считать токены и стоимость выполнения.

## Архитектура MVP

В проекте предполагаются:

- Flask API;
- PostgreSQL;
- 3 воркера;
- протоколы `TrackerProtocol` и `ScmProtocol`;
- mock-реализации `MockTracker` и `MockScm` для первого этапа.

### Воркеры

- `Worker 1` забирает задачи из трекера и комментарии из PR.
- `Worker 2` выполняет задачи агента, работает с git и PR, считает токены.
- `Worker 3` отправляет результат обратно в трекер.

## Структура проекта

- `src/backend` — основной код приложения;
- `instration/project.md` — текущее ТЗ проекта;
- `instration/instruction.md` — инструкция по работе с задачами;
- `instration/tasks/` — атомарные задачи для реализации;
- `AGENTS.md` — правила для агентной разработки.

## Локальный запуск базы

- Перед стартом API или воркеров подготовьте схему MVP командой `make bootstrap-db`.
- Команда создает только таблицы `tasks` и `token_usage` и безопасна для повторного запуска.
- Для прямого запуска через `uv` используйте `uv run heavy-lifting-bootstrap-db`.
- При необходимости можно переопределить подключение на один запуск: `uv run heavy-lifting-bootstrap-db --database-url sqlite+pysqlite:///./dev.db`.
