# Pre-Commit Checks Skill

## Purpose

Этот skill задает обязательный минимум проверок перед коммитом.

## Rules

- Перед коммитом кодовой задачи `DEV` обязан запустить предкоммитные проверки.
- Для текущего проекта обязательный минимум:
  - `make lint`
  - `make typecheck`
- Результат проверок должен быть записан в `instration/tasks/taskN_progress.md`.
- Если проверка неприменима к задаче или временно не может быть запущена, это должно быть явно объяснено в `taskN_progress.md`.
- Если проверки падают, коммит делать нельзя, пока проблема не исправлена или пока нет явно зафиксированной причины отклонения от правила.

## Recommended Pattern

- Для кодовых задач запускай перед коммитом:

```bash
make lint
make typecheck
```

- Затем фиксируй в `taskN_progress.md`:
  - какие команды запускались;
  - прошли ли они успешно;
  - если не запускались, почему.

## Examples

### Example 1: backend code change

Изменения затронули `src/backend/...`.

Перед коммитом нужно выполнить:

```bash
make lint
make typecheck
```

И записать результат в `taskN_progress.md`.

### Example 2: documentation-only task

Изменения затронули только `instration/...` и `AGENTS.md`.

В этом случае можно не запускать `make lint` и `make typecheck`, но в `taskN_progress.md` нужно явно записать, что задача документационная и код проекта не менялся.

### Example 3: partial failure

Если `make lint` прошел, а `make typecheck` нет, коммит не создается.
Сначала нужно исправить проблему или явно задокументировать, почему задача временно не может соответствовать правилу.

## When To Use

- Используй этот skill перед каждым коммитом кодовой задачи.
- При изменении правил локальных проверок обновляй этот skill вместе с `AGENTS.md` и конфигурацией `DEV`.
